En pro de aplicar la lógica de programación, vamos a hacer el reto de https://retosdeprogramacion.com/ejercicios usando tres lenguajes que considero clave
- **Python** . Es el lenguaje rey, en el se sustenta casi toda la programación es una navaja suiza y el ciencia de datos brilla me va a servir de mucho
- **Javascript** . Es el lenguaje para publicar en la Web, tiene frameworks potentes y vale la pena aprenderlo para que el código no solo quede en la terminal
- **Rust** . No conozco mucho a este lenguaje, es de bajo nivel requiere un poco de ciencia, pero se que es un lenguaje del futuro tiene mucho potencial y veamos que pasa
### Reglas
1. No puedo pegar directamente código de la IA. La IA es para Auditar
2. Cada día se cumplirá un reto así hasta llegar al 30vo día
3. Se rotara cada día con el lenguaje, si hoy hice python mañana será js y pasado rust, a menos que el reto sea muy especifico y requiera un lenguaje especifico
4. No se podrá fallar mas de 2 días en el reto, debe ser consecutivo
5. Se documentara el avance en obsidian, de cada reto excepto si el programa es demasiado sencillo
6. Se manejaran comentarios en el código
7. Se usara ramas de github y un repo de github donde se subirá el avance
8. Todo estara corriendo en WSL en Ubuntu
#### Consideraciones
- Sino aparece la carpeta de Linux en algo prueba con 
```
\\wsl$
```
_(O también puedes probar con `\\wsl.localhost`)_
- Para este reto de scripts de lógica de 30 días te va a funcionar de 10 y de forma súper cómoda. Pero si en el futuro (cuando estés en el Día 13 o 14 con Rust compilando cosas más pesadas) notas que GitHub Desktop se pone un poco lento para leer los cambios, el truco definitivo es abrir tu VS Code desde la terminal de Linux con `code .` usando la extensión **WSL - Remote** y manejar los commits desde la pestaña de Git integrada de VS Code. ¡Ahí va a la velocidad de la luz!
